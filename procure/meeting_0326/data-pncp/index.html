    PNCP Web Search¶

    Purpose¶

    The Audesp data records procurement processes by modality but does not explicitly flag which law governs each process (Lei 8.666/93 vs. Lei 14.133/2021). To establish when each municipality first adopted the NLL, we queried the Portal Nacional de Contratações Públicas (PNCP), the federal platform where NLL-governed procurements must be published.

    A municipality's first PNCP publication serves as a proxy for NLL adoption.

    Method¶

    Script: pncp_adocao_por_municipio.py

    Queries the PNCP REST API for each of 645 SP municipalities.

    API details¶

    • Endpoint: https://pncp.gov.br/api/consulta/v1/contratacoes/publicacao
    • Parameters: dataInicial, dataFinal, codigoModalidadeContratacao, codigoMunicipioIbge
    • Constraint: API rejects date ranges > 1 year (HTTP 422)

    Query strategy¶

    For each municipality, the script searches:

    • 5 annual windows: 2021, 2022, 2023, 2024, 2025
    • 4 NLL-relevant modality codes: Concurso (3), Pregão Eletrônico (6), Dispensa (8), Inexigibilidade (9)

    This yields ~20 API requests per municipality (~8 seconds at 0.4s rate limiting).

    Resilience features¶

    • Rate limiting: 0.4s between requests
    • Timeout: 60s per request
    • Retries: 3 attempts with 2s wait
    • Resumable checkpoint: checkpoint_municipios.json saves progress per municipality, enabling restart after interruption

    Municipality IBGE codes are fetched from the IBGE API (servicodados.ibge.gov.br/api/v1/localidades/estados/35/municipios).

    Data obtained¶

    Output: adocao_14133_por_municipio_sp.csv (645 rows)

    Field Description
    cod_ibge IBGE 7-digit municipality code
    municipio Municipality name
    data_primeira_14133 Date of first NLL publication
    ano_adocao Year of first publication
    n_publicacoes Total NLL publications found
    modalidade_primeira Modality of first NLL procurement
    status "ok" or "sem_publicacao"

    Limitations¶

    1. Compliance lag: Not all NLL-governed procurements may appear in PNCP immediately. Adoption dates represent a lower bound.
    2. Future dates: The API occasionally returns future dates (e.g., 2025-12-10 for Altair) — likely data entry errors.
    3. API instability: The PNCP API occasionally returns HTTP 422 for valid date ranges, requiring retry logic.
    4. Coverage gaps: Municipalities that adopted NLL but did not publish in PNCP would be misclassified as never-treated.

    5. Roadmap for Top-5¶

    Seven concrete actions, ordered by priority. Each with difficulty, expected impact, and dependencies.

    Action 1: Implement the NLL Staggered DiD¶

    Priority: #1 --- Critical

    What to do. Build municipal adoption panel from fundamento_dispensa (identifies old vs. new law for each dispensa). Estimate event study on irregularity tipo shares --- both the 37-type LLM codebook and the coarser 6-type keyword categories. Report pre-trends and compositional shifts. Key test: does the direction of change for each tipo match the Santos & Souza predictions?

    Predicted change Tipo Direction
    Cover bidding simulacao :arrow_down:
    Shell companies in convite empresa_fantasma :arrow_down:
    Fracionamento at old thresholds fracionamento :arrow_down: shift :arrow_right:
    Specification rigging direcionamento :arrow_right: or :arrow_up:
    Fabricated emergencies emergencia_fabricada :arrow_up:
    Post-award extraction aditivo_irregular :arrow_up:
    • Difficulty: Medium (data ready, design articulated, needs coding + estimation)
    • Impact: Critical --- this is the paper's most promising test of its core claim (scheme substitution in response to institutional change)
    • Dependencies: LLM extraction Batch 1 complete; build_consulta_llm.py run to normalize

    Action 2: Report New Full-Panel Behavioral Outcomes for the Relator IV¶

    Priority: #2 --- Substantial

    What to do. Run the newly implemented Q4 spending concentration, repeat-contractor share, competition index, and payment gap outcomes. Report reduced-form coefficients. If any are significant at N ~ 8,000, this transforms the relator IV from "suggestive" to "credible."

    Outcome Source Expected N Why promising
    Q4 spending share despesa ~8,000 Full panel; novel; gaming signal
    Repeat-contractor share despesa ~7,400 Full panel; incumbency entrenchment
    Competition index licitacao+bid ~538 Theory-driven composite
    Payment timing gap contract+empenho ~550 Execution-stage measure
    • Difficulty: Low (code already written, needs execution on data machine)
    • Impact: Substantial if any outcome moves; critical for keeping relator IV as pillar
    • Dependencies: Build outputs must be generated from raw data

    Action 3: Complete LLM Validation¶

    Priority: #3 --- Critical

    What to do. Manually code 100 irregularity allegations at fine-grained type level. Compute Cohen's kappa and confusion matrix against GPT-4o-mini. Test prompt robustness (3--5 prompt variants). Report results in Section 6.3.

    Target metrics:

    • Cohen's kappa > 0.7 at 6-type level -> confident
    • Cohen's kappa 0.5--0.7 -> usable with caveats

    • Cohen's kappa < 0.5 -> retreat to coarser categories

    • Difficulty: Medium (requires legal expertise for coding; ~40 hours of RA time)

    • Impact: Critical --- without validation, the entire scheme classification is an assumption
    • Dependencies: None (can be done in parallel with everything else)

    Action 4: Expand Tier 4 via Name-Matching Refinement¶

    Priority: #4 --- Substantial

    What to do. Implement the 4-step narrowing procedure:

    1. Municipality filter (restrict firm matches to same municipality)
    2. Time window (firm active during court-referenced period)
    3. Proc-number cross-reference (match extracted proc numbers to despesa)
    4. LLM extraction from sentenca text (~1,270 API calls)

    Target: Expand from 56 to 200+ process-linked cases.

    • Difficulty: Medium-high (~1,270 API calls; data engineering for steps 1--3)
    • Impact: Substantial --- makes bid-signature regressions credible rather than illustrative
    • Dependencies: Steps 1--3 are pure data operations; Step 4 requires API budget

    Action 5: Implement McCrary Bunching Test for H1¶

    Priority: #5 --- Substantial

    What to do. Apply Cattaneo, Jansson, and Ma (2020) local polynomial density estimator to contract-value distribution around dispensa/convite thresholds. Compare density discontinuity between fracionamento-revealed municipalities and controls.

    • Difficulty: Low (standard procedure, R package rddensity or Python equivalent; data ready in licitacao.parquet)
    • Impact: Substantial --- threshold bunching is the cleanest, most visual test of any single hypothesis; H1 is the only hypothesis where the mechanism is deterministic
    • Dependencies: None (can be run immediately)

    Action 6: Resolve Measurement Error in LLM Classifier¶

    Priority: #6 --- Depends on kappa

    What to do. After completing Action 3, implement either SIMEX correction (if classification error rates are estimable from the validation exercise) or a bootstrap sensitivity analysis that draws from the confusion matrix to assess how misclassification affects compositional estimates.

    • Difficulty: High (requires methodological care; no off-the-shelf solution for multinomial classification error in panel outcomes)
    • Impact: Marginal to substantial depending on error rates
    • Dependencies: Action 3 (validation results needed first)

    Action 7: Restructure the Paper¶

    Priority: #7 --- Substantial for readability

    What to do. Move network analysis (Section 9), enforcement funnel (Section 8.4), and cross-source validation (Section 10) to appendix. Main text structure:

    1. Introduction (policy-motivated, per Henrik's comment)
    2. Framework (5 lenses, streamlined)
    3. Data + Classification + Validation
    4. Relator IV results
    5. NLL results
    6. Discussion

    Target: 40--50 pages (down from current ~80).

    • Difficulty: Low (editorial, not analytical)
    • Impact: Substantial for readability and reviewer goodwill
    • Dependencies: Actions 1--3 complete (so main results are known before restructuring)
